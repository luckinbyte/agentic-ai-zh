# 翻译规范(Translation Guide)

> 本文档是所有翻译 agent 的统一指令。翻译任一章节前,请先完整阅读本规范与 [GLOSSARY.md](GLOSSARY.md),并严格遵循。

## 1. 总则

- **目标语言**:简体中文。
- **风格**:意译为主,通顺、自然、符合中文技术写作习惯;在保证可读性的前提下忠实于原意。**严禁机翻腔**。
- **技术准确**:术语、算法、公式、架构描述必须准确,不得臆造。
- **完整性**:完整翻译正文,**不省略、不增撰**。仅可删去纯排版噪音(页码、页眉作者名 "H. Roitman"、重复出现的章名页眉)。
- **输出**:用译文**完整覆盖**目标章节文件(`chapters/.../<chapter>.md`),替换原骨架占位。文件以中文 `#` 一级标题开头。

## 2. 术语处理

- **通用英文缩写一律保留英文不译**(首现时可加中文释义):RAG、MCP、A2A、LLM、PPO、GRPO、DPO、SFT、RLHF、PEFT、LoRA、MoE、BPE、API、SDK、JSON、HTTP、SSE、URL、TF-IDF、BM25、DPR、SPLADE 等。
- **其他专业术语首现时写「中文(English)」**,之后用中文。例:`检索增强生成(Retrieval-Augmented Generation, RAG)`首次,之后用「RAG」。
- 必须查阅 [GLOSSARY.md](GLOSSARY.md),**全书术语保持一致**。若发现术语表未收录且重要的术语,在返回结果中上报,便于统一。
- 人名(如 Haggai Roitman、Sutton)、论文/产品名(LangGraph、AutoGen、CrewAI、MemGPT)、数据集/基准名保留原文。
- 参考文献编号(如 `[128]`)原样保留。

## 3. 标题与编号

- 保留原书的**章节编号体系**(如 `16.1`、`16.1.1`)。
- **合并 PDF 中分行的编号与标题**:原文常把编号和标题拆成两行(如 `16.1` 一行、`Motivation and Problem Statement` 一行),译为 `## 16.1 动机与问题陈述`。
- 层级:`#` = 章标题,`##` = 节(L3),`###` = 子节(L4)。与原书层级一致。
- 骨架文件已含英文标题结构,仅供参照层级;**最终标题用中文**(可带编号)。

## 4. 数学公式(关键)

PDF 提取的公式常有破损,**必须据上下文重建为正确的 LaTeX**:

- 求和符号被误提为 `X` → 还原为 `\sum`。例如 `X_{d∈D}` → `\sum_{d \in D}`。
- 下标丢失:`PMθ` → `P_{M_\theta}`;`Pret` → `P_\text{ret}`。
- 上下标、希腊字母、矩阵、分式按原意重建:`Mθ` → `M_\theta`、`D = {d1,...,dN}` → `D = \{d_1, \dots, d_N\}`。
- **行内公式**用 `$...$`,**独立公式**用 `$$...$$`。
- 公式编号(如 `(16.1)`)保留,写作 `$$ ... \tag{16.1} $$`。
- 不确定时,依据上下文语义给出最合理的 LaTeX,不要照抄破损符号。

## 5. 代码

- **代码块保留英文原文**,不翻译代码。保留或补全语言标注(``` ```python ` ```、``` ```bash ` ``` 等)。
- 仅在**关键处**追加中文注释(用 `#`),解释易误解处;不逐行注释,不改动逻辑。
- 行内代码(变量名、函数名、参数名)用反引号包裹,保留英文。

## 6. 图片

- 译文位置应与原文插图位置对应。
- 该章可用的插图文件会在任务中列出(相对当前 md 文件的路径,形如 `../../images/<part>/<chapter>/xxx.png`)。
- 用 `![中文图注](../../images/.../xxx.png)` 引用,**图注用中文**(据原文 Figure 说明翻译)。例:`![图 16.1:RAG 完整流水线](../../images/...)`。
- 若原文提到某 Figure 但**没有对应的提取图片**(部分矢量图未被提取),则在相应位置用引用块给出**中文文字描述**,并标注 `(参见原文 pX 图)`。
- 可用图片数量与原文 Figure 数量未必相等,尽力按出现顺序与页码对应;无法对应的图在章末列出说明。

## 7. 其他 markdown 元素

- **列表**(有序/无序)、**表格**、**引用块** `>` 一律保留结构,内容译为中文。
- **加粗/斜体**保留。
- **超链接**保留,链接文字译中文,URL 不变。
- 段落间空行保留,确保 markdown 正确渲染。

## 8. 不要做的事

- ❌ 不要保留 `> 本章待翻译` 之类的骨架占位文字。
- ❌ 不要保留页眉 "H. Roitman"、孤立页码、PDF 提取产生的多余空行。
- ❌ 不要翻译代码、URL、产品名、参考文献编号。
- ❌ 不要逐字死译导致中文不通顺。
- ❌ 不要添加原文没有的个人评论或内容。

## 9. 产出检查(写文件前自检)

1. 是否覆盖了原文全部小节,无遗漏?
2. 公式是否已重建为合法 LaTeX,无残留 `X`(求和)、破损下标?
3. 章节编号与标题是否已合并、层级正确?
4. 图片是否已用正确相对路径引用?
5. 术语是否与 GLOSSARY 一致?
