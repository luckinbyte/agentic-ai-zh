# 第 4 章 语言模型的强化学习基础

监督微调(Supervised Fine-Tuning, SFT)教模型去模仿示范数据,但模仿存在天花板:模型永远无法超越其训练数据的质量。强化学习(Reinforcement Learning, RL)打破了这一壁垒。通过生成新文本、接收奖励反馈、并朝着更高奖励的行为更新模型,经过 RL 训练的模型能够发现任何人类示范者都未曾写出的策略——从而产出更有帮助、更准确、且更契合人类偏好的输出 [9]。这正是每一个前沿模型背后的机制:GPT-4 [23]、Claude、Llama-3 [25] 以及 DeepSeek-R1 [15] 都在 SFT 之后应用 RL,以此作为把一个能力强但缺乏引导的模型转变为对齐助手的关键步骤。

## 4.1 LLM 中强化学习的两种范式

面向语言模型的 RL 方法可归为两大范式,各自适用于不同目标:

**范式一:通过人类偏好进行对齐(RLHF/DPO)。** 将 RL 应用于 LLM 的最初动机是对齐(alignment)——让模型变得有帮助、无害、诚实。基于人类反馈的强化学习(Reinforcement Learning from Human Feedback, RLHF)[9, 174, 175] 从成对的人类评判("哪个回答更好?")中训练一个奖励模型(reward model),随后优化策略以最大化该学得的奖励。DPO [10] 则通过完全移除奖励模型对此加以简化,把偏好直接转化为一个监督损失。两种方法都能产出遵循指令并尊重安全约束的对齐助手。

**范式二:通过可验证奖励进行能力增强(RLVR)。** 更近期地,RL 不仅用于对齐,还被用于教授新能力——尤其是推理、数学和代码生成。此时奖励不再来自人类偏好,而来自可验证的结果:模型是否给出了正确答案?代码是否通过了全部测试?DeepSeek-R1 [15] 证明,采用基于规则奖励(格式正确性 + 答案准确率)的 GRPO 可以在完全没有人类偏好数据的情况下,训练出具备复杂思维链(chain-of-thought)推理能力的模型。这一范式——基于可验证奖励的强化学习(Reinforcement Learning from Verifiable Rewards, RLVR)——如今已成为构建推理模型和智能体系统的主流方法。

**共同基础。** 尽管目标不同,两种范式共享同一套核心机制:

- 一个自回归地生成文本的策略 $\pi_\theta$(即 LLM)
- 一个奖励信号 $r(x, y)$(从偏好中学习,或从验证中计算得到)
- 一个相对于参考策略的 KL 约束,以防止退化解(degenerate solutions)
- 策略梯度优化(PPO 或 GRPO),用以将模型朝更高奖励的方向更新

本部分后续各章将逐一详细展开这些组件。

## 4.2 将文本生成建模为 MDP

使 RL 能够应用于语言模型的关键洞见,是把自回归生成重新表述为一个马尔可夫决策过程(Markov Decision Process, MDP):

**LLM 即智能体的类比。** 把 LLM 想象成一个一次写出一个 token 的智能体。在每一步,它观察到目前为止已写下的全部内容(状态),选择下一个词(动作),页面随之增长一个 token(转移)。当回复完成时,评判者对其打分(奖励)。目标是:学习一种能稳定获得高分的写作策略(即策略)。

形式上,文本生成的 MDP 如下:

- **状态** $s_t = (x, y_1, \dots, y_{t-1})$:提示与迄今为止已生成的全部 token 的拼接。
- **动作** $a_t \in \{1, \dots, |V|\}$:从词表中选取下一个 token(通常有 32K–128K 种选择)。
- **转移** $P(s_{t+1} \mid s_t, a_t)$:确定性的——只需追加所选 token,无环境随机性。
- **奖励** $r$:通常只在生成结束时给出(稀疏)。对 RLHF 而言是奖励模型的打分;对 RLVR 而言是最终答案的正确性。
- **策略** $\pi_\theta(a_t \mid s_t)$:LLM 的下一 token 概率分布——正是 softmax 输出已经计算出的量。
- **折扣** $\gamma = 1.0$:回合是有限的(一次回复),因此无需折扣。

这一映射之所以强大,是因为 LLM 本身就是一个策略——其 softmax 输出为每个状态定义了 $\pi_\theta(a_t \mid s_t)$。我们无需再单独构建一个策略网络;只需调整权重 $\theta$,使模型对那些能获得更高奖励的 token 序列赋予更高概率。

## 4.3 RLHF 流水线

经典的 RLHF 流水线 [9] 包含四个阶段:

1. **监督微调(SFT)**:在高质量示范数据上训练基座模型,得到一个能够遵循指令的策略 $\pi_{\text{SFT}}$。
2. **奖励模型训练**:收集人类偏好比较(对同一提示有 $y_w \succ y_l$),并使用 Bradley-Terry 目标训练奖励模型 $R_\phi(x, y)$。
3. **RL 优化**:以奖励模型为信号,在相对于 $\pi_{\text{SFT}}$ 的 KL 约束下,通过 PPO 或 GRPO 优化策略。
4. **评估与迭代**:评估对齐后的模型,收集新的失败案例,并反复迭代。

对于 RLVR(推理/智能体训练),阶段 1–2 被替换:SFT 模型在推理轨迹上训练,奖励模型则由一个验证器(verifier)取代(例如检验数学正确性)。阶段 3 保持不变——仍是针对奖励信号的 PPO 或 GRPO 优化。

**LLM 的 RL 与经典 RL 的区别。** LLM 场景在若干重要方面不同于经典 RL:

- **确定性转移**:"下一状态"仅仅是已有 token 的拼接——不存在随机环境。
- **稀疏奖励**:反馈通常只在生成结束时给出一次(结果奖励 / outcome reward),或在关键步骤给出(过程奖励 / process reward)。
- **巨大的动作空间**:每一步都有 32K–128K 种可能的 token,但探索是通过温度采样隐式进行的。
- **KL 锚点**:LLM 的 RL 受约束以保持接近 SFT 策略,以牺牲部分探索为代价来防止奖励黑客(reward hacking)。
- **无需价值函数**:GRPO 完全移除了评论家(critic)网络,转而采用奖励的组相对归一化。

这些差异解释了为何 PPO 和 GRPO 在 LLM 上压倒性地优于 DQN 式方法。

## 4.4 本部分路线图

接下来的章节将构建完整的 RL-for-LLMs 工具集:

1. **PPO(第 5 章)** —— 截断式替代目标(clipped surrogate objective)、用于优势估计的 GAE、评论家网络,以及完整的 RLHF 训练循环。这是 GPT-4 和 Claude 背后的主力方法。
2. **DPO(第 6 章)** —— 通过将偏好转化为对比式监督损失而完全绕过 RL。比在线 RL 更简单,但灵活性更低。
3. **GRPO(第 7 章)** —— DeepSeek 提出的无评论家算法,采用组级别奖励归一化。它是 DeepSeek-R1 背后的方法,也是推理模型训练的主流选择。
4. **偏好优化变体(第 8 章)** —— 在线 DPO、KTO、Best-of-N,以及方法选择指南。
5. **奖励建模(第 9 章)** —— Bradley-Terry 模型、过程奖励与结果奖励、面向 RLVR 的基于规则的奖励,以及多目标组合。
6. **SFT 最佳实践(第 10 章)** —— 序列打包、对话模板、数据混合,以及 SFT 质量如何决定 RL 的天花板。
7. **系统工程(第 11 章)** —— 大规模分布式训练:并行策略、生成与训练的解耦,以及支撑数百块 GPU 的基础设施。
